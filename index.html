Open Collections Protocol

Publish and discover collections like the web

Open Collections Protocol is a lightweight, web-native way to publish and discover collections as ordinary web resources.

Collection Manager and Collection Browser help teams edit, publish, and view collections as standalone tools or embeddable web components.

Collections should be easier to publish

Open Collections Protocol is built for institutions and community teams that want practical publishing, not platform lock-in.

You can host published collections with common storage providers, including Git-based platforms and S3-compatible object storage, without relying on one central system.

A simple publishing model

1. Publish a collection manifest

Give the collection a stable web URL and a machine-readable manifest.

2. Link item and media URLs

Each item and asset has a stable address that tools and people can reference.

3. Add domain discovery (optional)

Publish a Domain Collections Discovery (DCD) document to advertise collections from a domain.

4. Reuse across the ecosystem

Indexers, browsers, portals, and aggregators can all use the same published surface.

Many tools, one shared web surface

LinkedCollections is the architectural model behind the protocol, and Domain Collections Discovery (DCD) supports domain-level discovery.

Publisher
Publishes collection manifests, item resources, and media to the open web.
Collection Manager
Create, edit, organize, and publish collections.
Registrator
Register, validate, or curate collection endpoints.
Indexer
Discover and organize collections for search and aggregation.
Collection Browser / TimeMap
Explore collections through general browsing or interpretive interfaces.

Collection Manager and Collection Browser can work both as standalone tools and as embeddable components inside websites, portals, and custom applications.

Open, portable, and web-native

Publish once on the open web, keep control of your own domain and hosting, and let many tools interoperate with the same collection resources.
