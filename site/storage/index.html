Storage Providers

Open Collections Protocol does not require one hosting platform. Publish collections using storage choices that fit your team while keeping stable public URLs.

What every storage setup must provide

  • a stable collection.json URL
  • stable item and media URLs
  • optional DCD discovery file at /.well-known/collections.json
  • enough durability for long-term collection access

Git-based providers

Good for versioned, file-first publication workflows and small/medium collections.

  • Examples: GitHub, GitLab, Forgejo, Gitea
  • Strengths: version history, review workflows, transparent file changes
  • Typical use: curated public collections and iterative publishing

S3-compatible object storage

Good for larger media workloads and scalable asset delivery.

  • Examples: AWS S3, Cloudflare R2, Backblaze B2 (S3), MinIO, Wasabi
  • Strengths: scalability, CDN-friendly asset hosting, media-heavy collections
  • Typical use: larger institutional datasets and high-volume media delivery

Common mixed model

  • Store manifests and metadata files in Git-based publishing
  • Store large media in S3-compatible buckets
  • Expose both through stable URLs
