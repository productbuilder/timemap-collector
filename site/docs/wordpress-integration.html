WordPress integration

WordPress is an adoption layer, CMS host environment, and configuration/routing layer. Open Collections Protocol remains the public contract.

First plugin direction

A practical first plugin is Open Collections for WordPress, focused on embedding Collection Manager, managing settings, and publishing protocol-facing outputs.

  • embed Collection Manager in admin/pages
  • configure collection roots and output routes
  • publish manifest, item, and media resources
  • optionally expose DCD at /.well-known/collections.json

Protocol outputs to preserve

  • collection.json manifest URL
  • stable item detail URLs
  • stable media URLs
  • optional DCD file at /.well-known/collections.json
  • optional registry export in later phases

The public contract should be these outputs, not internal WordPress storage structures.

Repository scaffold status

A first plugin scaffold is now in the repository at integrations/wordpress/open-collections/. It provides bootstrap/setup, admin settings, and initial embed/config stubs.

Roadmap

  • Phase 1: Publishing + Manager embed
  • Phase 2: Browser embed + better UX
  • Phase 3: Optional registry participation
  • Phase 4: Advanced integrations and provider helpers
