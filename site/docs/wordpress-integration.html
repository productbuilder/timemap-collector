WordPress integration

WordPress is an adoption path, not the architecture itself. LinkedCollections and the Open Collections Protocol web contract remain the core.

Where WordPress helps

  • familiar CMS workflows for non-technical publishing teams
  • plugin-based integration into existing institutional websites
  • quick path to publishing collection manifests and item pages

Protocol-facing outputs to preserve

  • collection.json manifest URL
  • stable item and media URLs
  • DCD file at /.well-known/collections.json (when applicable)
  • optional registry export for Registry participation

The key rule: WordPress internals should not become the public protocol contract.

Integration modes

  • Publishing plugin for manifest/item/media outputs
  • Embedded Collection Browser in public pages
  • Embedded Collection Manager in admin workflows
  • Optional lightweight Collection Registry role

Recommended sequence

  1. Start with publishing outputs and stable URLs.
  2. Add DCD for domain discovery.
  3. Embed Collection Browser for presentation.
  4. Add Manager and Registry capabilities as needed.
