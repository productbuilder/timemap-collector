Components and embedding

Collection Manager and Collection Browser can be used as standalone applications or embedded components inside websites, portals, and CMS workflows.

Embedding goals

  • reuse core publishing/browsing behavior across multiple sites
  • keep protocol outputs independent from host platform internals
  • configure behavior through clear attributes/options

Current implementation pointers

  • Collection Manager app README
  • Collection Browser app README
  • Demo page with embedded Collection Manager

Recommended embedding checklist

  • define allowed sources/providers in host context
  • set clear read/write permission boundaries
  • validate generated URLs and manifests against protocol expectations
